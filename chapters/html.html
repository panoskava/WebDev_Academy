Chapter 2: HTML - Structure of Web Pages

2.1 What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It describes the content and structure of a web page using elements represented by tags. HTML is NOT a programming language - it's a markup language that tells browsers how to structure content.

Key Concepts

  • Content vs. Presentation: HTML describes WHAT content is (a heading, a paragraph, a list), NOT how it looks. Appearance is controlled by CSS.
  • Tags vs. Elements: A tag is the markup (like <p>), an element is the complete unit including content (<p>content</p>)
  • Nesting: Elements can contain other elements, creating a tree structure
  • Whitespace: Most whitespace in HTML is collapsed into a single space
Basic Tag Syntax <!-- Opening tag, content, closing tag -->
<element>content</element>

<!-- Example: -->
<p>This is a paragraph</p>
Complete HTML Document Structure <!DOCTYPE html> <!-- Declares HTML5 -->
<html lang="en">
  <head>
    <!-- Metadata about the page -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <!-- Visible page content -->
    <h1>Hello, World!</h1>
    <p>Welcome to my website.</p>
  </body>
</html>

HTML Document Parts Explained

1. <!DOCTYPE html> - Document Type Declaration
  • Purpose: Tells the browser which version of HTML the page is written in
  • Required: Must be the FIRST line of every HTML document
  • Not a tag: It's a declaration, not an HTML element (no closing tag)
  • Case insensitive: <!DOCTYPE html>, <!doctype html>, <!DOCTYPE HTML> all work
  • Without it: Browser enters "quirks mode" and may render incorrectly
<!-- HTML5 (current standard - use this!) -->
<!DOCTYPE html>

<!-- Old HTML 4.01 (don't use anymore) -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN">
2. <html> - Root Element
  • Purpose: The root (top-level) element that contains ALL other elements
  • Required: Every HTML document must have exactly ONE html element
  • lang attribute: Should specify the document's language for accessibility and SEO
  • Contains: Only <head> and <body> as direct children
<html lang="en"> <!-- English -->
<html lang="el"> <!-- Greek -->
<html lang="en-US"> <!-- American English -->
3. <head> - Document Metadata Container

The head contains information ABOUT the page, NOT visible content.

  • <meta charset="UTF-8">: Character encoding - MUST be first element in head for proper text rendering
  • <meta name="viewport">: Controls page dimensions and scaling on mobile devices
  • <meta name="description">: Page description shown in search engine results
  • <meta name="keywords">: Keywords for SEO (less important today)
  • <meta name="author">: Author of the document
  • <title>: Page title shown in browser tab and search results (REQUIRED)
  • <link>: Links external resources (CSS stylesheets, favicon, fonts)
  • <style>: Internal CSS styles (embedded in the page)
  • <script>: JavaScript code or links to external JS files
  • <base>: Base URL for all relative URLs in the document
<head>
  <!-- Character encoding - MUST be first -->
  <meta charset="UTF-8">

  <!-- Viewport for responsive design -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- SEO meta tags -->
  <meta name="description" content="Learn HTML and CSS">
  <meta name="author" content="Your Name">

  <!-- Page title (shown in browser tab) -->
  <title>My Web Page</title>

  <!-- External CSS stylesheet -->
  <link rel="stylesheet" href="styles.css">

  <!-- Favicon (browser tab icon) -->
  <link rel="icon" href="favicon.ico">
</head>
4. <body> - Document Body

Everything the user sees goes inside the body element.

  • Contains: ALL visible content - text, images, links, videos, forms, etc.
  • Structure: Should be organized with semantic elements (header, nav, main, footer)
  • Only one: Each HTML document has exactly ONE body element
  • Events: Can have event attributes like onload, onunload
<body>
  <header>
    <nav>...navigation...</nav>
  </header>

  <main>
    <article>...main content...</article>
  </main>

  <footer>...footer content...</footer>

  <!-- Scripts at the end for better performance -->
  <script src="app.js"></script>
</body>

Important Rules for HTML Documents

  • Order matters: DOCTYPE ‚Üí html ‚Üí head ‚Üí body (always this order)
  • Nesting: Elements must be properly nested (close inner tags before outer tags)
  • Case insensitive: <HTML> = <html> = <Html> (but lowercase is convention)
  • Quotes: Attribute values should be in quotes (single or double)
  • Self-closing tags: Some elements have no closing tag: <meta>, <link>, <br>, <img>, <input>

2.2 Block vs Inline Elements

Understanding the difference between block and inline elements is crucial for layout:

Block Elements

  • Take up the full width available
  • Always start on a new line
  • Stack vertically
  • Examples: <div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>, <section>, <article>

Inline Elements

  • Take up only as much width as necessary
  • Do NOT start on a new line
  • Flow horizontally with text
  • Examples: <span>, <a>, <strong>, <em>, <img>, <br>
Block vs Inline Example <!-- Block elements stack vertically -->
<div>This takes full width</div>
<div>This starts on a new line</div>

<!-- Inline elements flow horizontally -->
<p>This is <strong>bold</strong> and <em>italic</em> text.</p>

2.3 Text Elements: Headings and Paragraphs

HTML provides 6 levels of headings and paragraph elements for text content. Proper use of headings is crucial for both accessibility (screen readers use them for navigation) and SEO (search engines use them to understand page structure).

Headings Hierarchy

Headings create a document outline. Think of them like chapters and sections in a book:

  • <h1> - Main page title (use only ONE per page)
  • <h2> - Major sections (like chapter titles)
  • <h3> - Subsections within h2
  • <h4> to <h6> - Further subdivisions
Headings (h1-h6) <h1>Main Title - Largest, most important</h1>
<h2>Major Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

<!-- Best Practice: Use only ONE h1 per page -->
<!-- Headings should follow a logical order (h1 ‚Üí h2 ‚Üí h3) -->
‚ö†Ô∏è Common Mistake:

Don't skip heading levels! Going from <h1> directly to <h4> is bad practice. Always follow the logical order: h1 ‚Üí h2 ‚Üí h3 ‚Üí h4

Paragraphs and Text Containers

The <p> element defines a paragraph. Browsers automatically add margin before and after paragraphs.

Paragraphs and Line Breaks <p>This is a paragraph. Paragraphs are block elements
that add spacing before and after the text.</p>

<p>For a line break within a paragraph,<br>
use the br element (self-closing).</p>

<!-- Horizontal rule for visual separation -->
<hr>

Important Notes

  • <br> creates a line break WITHOUT starting a new paragraph (use sparingly)
  • <hr> creates a horizontal line (thematic break between sections)
  • Whitespace collapse: Multiple spaces/newlines in HTML code become a single space when displayed

2.4 Text Formatting Elements

HTML provides elements for emphasizing and formatting text. It's important to understand the difference between semantic elements (which convey meaning) and presentational elements (which only affect appearance).

Semantic vs Presentational Elements

Semantic (Use These!) Presentational (Avoid) Visual Result
<strong> <b> Bold text
<em> <i> Italic text

Why does this matter? Screen readers will emphasize <strong> and <em> content, but ignore <b> and <i>. Search engines also give more weight to semantic elements.

Text Formatting Elements <!-- Semantic emphasis (preferred) -->
<strong>Important text (bold)</strong>
<em>Emphasized text (italic)</em>
<mark>Highlighted text</mark>

<!-- Visual formatting (use sparingly) -->
<b>Bold (no semantic meaning)</b>
<i>Italic (no semantic meaning)</i>
<u>Underlined text</u>
<s>Strikethrough text</s>

<!-- Subscript and Superscript -->
H<sub>2</sub>O <!-- Water formula -->
x<sup>2</sup> <!-- x squared -->

<!-- Code and preformatted text -->
<code>let x = 5;</code> <!-- Inline code -->
<pre>Preformatted text
preserves whitespace</pre>

Other Useful Text Elements

  • <blockquote> - For longer quotations (block-level)
  • <q> - For short inline quotations (adds quotation marks)
  • <cite> - For citing sources/titles
  • <abbr> - For abbreviations (use with title attribute)
  • <small> - For fine print, copyright notices
  • <del> and <ins> - For showing deleted and inserted text
Quotations and Citations <!-- Block quotation -->
<blockquote cite="https://source.com">
  <p>A long quotation goes here...</p>
  <footer>‚Äî <cite>Author Name</cite></footer>
</blockquote>

<!-- Inline quotation -->
<p>She said <q>Hello world</q> to the audience.</p>

<!-- Abbreviation with tooltip -->
<abbr title="HyperText Markup Language">HTML</abbr>

2.5 Links and Navigation

The anchor element <a> creates hyperlinks - the foundation of the web. Links are what make the "web" a web, connecting pages together.

Understanding the href Attribute

The href (Hypertext REFerence) attribute specifies the destination of the link:

  • Absolute URL: https://www.example.com/page.html - Full address including protocol
  • Relative URL: page.html or ../folder/page.html - Relative to current page
  • Root-relative: /images/photo.jpg - Relative to website root
  • Fragment: #section-id - Jump to element with that ID
Types of Links <!-- External link (to another website) -->
<a href="https://www.google.com">Visit Google</a>

<!-- Open in new tab (add rel for security) -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">New Tab</a>

<!-- Internal link (same website) -->
<a href="about.html">About Us</a>
<a href="/products/shoes.html">Shoes</a>

<!-- Anchor link (jump to section on same page) -->
<a href="#contact">Go to Contact Section</a>
...
<section id="contact">...</section>

<!-- Email link -->
<a href="mailto:info@example.com">Email Us</a>

<!-- Phone link (great for mobile) -->
<a href="tel:+1234567890">Call Us</a>

<!-- Download link -->
<a href="document.pdf" download>Download PDF</a>

Important Link Attributes

  • href - The destination URL (required)
  • target="_blank" - Opens link in new tab/window
  • rel="noopener noreferrer" - Security feature for external links (prevents the new page from accessing your page)
  • download - Prompts download instead of navigating
  • title - Tooltip text on hover
üí° Pro Tip:

Don't use "click here" as link text. Instead, use descriptive text that tells users where the link goes. Good: "Read our privacy policy". Bad: "Click here".

2.6 Lists

HTML supports three types of lists for organizing content. Using the correct list type is important for semantics and accessibility.

When to Use Each List Type

  • Unordered List (<ul>) - When order doesn't matter (ingredients, features, navigation)
  • Ordered List (<ol>) - When sequence matters (instructions, rankings, steps)
  • Description List (<dl>) - For term-definition pairs (glossaries, FAQs, metadata)
Unordered List (bullets) <ul>
  <li>Apple</li>
  <li>Banana</li>
  <li>Orange</li>
</ul>
Ordered List (numbered) <ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>

<!-- Start from different number -->
<ol start="5">...</ol>
Description List (definitions) <dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
Nested Lists <ul>
  <li>Fruits
    <ul>
      <li>Apple</li>
      <li>Banana</li>
    </ul>
  </li>
  <li>Vegetables</li>
</ul>

2.7 Images and Media

Images make web pages visually engaging. The <img> element is used to embed images, and it's a self-closing (void) element - it has no closing tag.

Required Attributes

  • src (source) - Path to the image file (required)
  • alt (alternative text) - Description of the image (required for accessibility)
‚úÖ Why alt text is crucial:
  • Screen readers read it aloud for visually impaired users
  • Displays when image fails to load
  • Helps search engines understand image content (SEO)
  • Leave alt="" empty for purely decorative images
Images <!-- Basic image (self-closing element) -->
<img src="photo.jpg" alt="Description of image">

<!-- With dimensions (helps prevent layout shift) -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Image as a link -->
<a href="products.html">
  <img src="product.jpg" alt="View Product Details">
</a>

<!-- Figure with caption (semantic) -->
<figure>
  <img src="chart.png" alt="Bar chart showing Q4 sales by region">
  <figcaption>Q4 Sales Data by Region</figcaption>
</figure>

Common Image Formats

  • .jpg/.jpeg - Best for photographs (lossy compression)
  • .png - Best for graphics with transparency
  • .gif - Simple animations, limited colors
  • .svg - Vector graphics, scales perfectly
  • .webp - Modern format, smaller file sizes
Video and Audio <!-- Video with controls -->
<video src="video.mp4" controls width="640">
  Your browser doesn't support video.
</video>

<!-- Audio -->
<audio src="song.mp3" controls></audio>

2.8 Tables

Tables are used to display tabular data (data that belongs in rows and columns). Tables should NOT be used for page layout - use CSS for that!

Table Structure Elements

  • <table> - Container for the entire table
  • <thead> - Groups header rows
  • <tbody> - Groups body content rows
  • <tfoot> - Groups footer rows (totals, summaries)
  • <tr> - Table Row
  • <th> - Table Header cell (bold, centered by default)
  • <td> - Table Data cell (regular content)
Complete Table Structure <table>
  <thead> <!-- Table header -->
    <tr> <!-- Table row -->
      <th>Name</th> <!-- Header cell -->
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody> <!-- Table body -->
    <tr>
      <td>Alice</td> <!-- Data cell -->
      <td>25</td>
      <td>New York</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>30</td>
      <td>London</td>
    </tr>
  </tbody>
</table>

2.9 Forms and User Input

Forms are essential for user interaction - login pages, search boxes, contact forms, checkouts, etc. The <form> element collects user input and sends it to a server.

Form Attributes

  • action - URL where form data is sent (server endpoint)
  • method - HTTP method: GET (data in URL) or POST (data in body)
  • enctype - Encoding type (use multipart/form-data for file uploads)

Common Input Types

HTML5 introduced many input types with built-in validation:

  • text, password, email, tel, url
  • number, range (slider)
  • date, time, datetime-local
  • checkbox, radio
  • file (for uploads)
  • hidden (not visible to user)
  • submit, reset, button
Complete Form Example <form action="/submit" method="POST">
  <!-- Text input -->
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>

  <!-- Email input -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">

  <!-- Password input -->
  <input type="password" name="pass" placeholder="Password">

  <!-- Textarea for longer text -->
  <textarea name="message" rows="4"></textarea>

  <!-- Dropdown select -->
  <select name="country">
    <option value="us">United States</option>
    <option value="uk">United Kingdom</option>
  </select>

  <!-- Radio buttons (choose one) -->
  <input type="radio" name="gender" value="m"> Male
  <input type="radio" name="gender" value="f"> Female

  <!-- Checkbox -->
  <input type="checkbox" name="agree"> I agree to terms

  <!-- Submit button -->
  <button type="submit">Submit</button>
</form>

2.10 Semantic HTML5 Elements

Semantic elements clearly describe their meaning to both browsers and developers:

Semantic Page Structure <header> <!-- Page/section header -->
  <nav> <!-- Navigation links -->
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>

<main> <!-- Main content (only one per page) -->
  <article> <!-- Self-contained content -->
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>

  <section> <!-- Thematic grouping -->
    <h2>Section Title</h2>
  </section>

  <aside> <!-- Sidebar content -->
    Related links...
  </aside>
</main>

<footer> <!-- Page/section footer -->
  <p>© 2024 My Website</p>
</footer>

Why Use Semantic Elements?

  • Accessibility: Screen readers understand page structure and can navigate by landmarks
  • SEO: Search engines better understand and index your content
  • Maintainability: Code is self-documenting and easier for other developers to understand
  • Styling: Provides meaningful hooks for CSS without needing extra classes

Semantic Elements Reference

Element Purpose
<header> Introductory content, typically contains logo and navigation
<nav> Navigation links (main menu, breadcrumbs)
<main> Main content of the page (only ONE per page)
<article> Self-contained content that could stand alone (blog post, news article)
<section> Thematic grouping of content, usually with a heading
<aside> Sidebar content, tangentially related to main content
<footer> Footer content, typically copyright, contact info, links
‚ö†Ô∏è Don't confuse:

<section> vs <div>: Use <section> when content has a heading and represents a thematic grouping. Use <div> only as a styling container with no semantic meaning.

2.11 HTML Attributes

Attributes provide additional information about elements. They are always specified in the opening tag and usually come in name="value" pairs.

Global Attributes (work on ANY element)

Attribute Purpose
id Unique identifier (must be unique on page, used for CSS/JS targeting)
class Reusable classifier (multiple elements can share, can have multiple classes)
style Inline CSS styles (avoid when possible, use external CSS)
title Tooltip text shown on hover
hidden Hides element from view (boolean attribute)
data-* Custom data attributes for JavaScript interaction

ID vs Class - When to Use Which?

  • id - Use for unique elements (header, main navigation, specific form)
  • class - Use for styling patterns that repeat (buttons, cards, alerts)
Attribute Examples <!-- id: Unique identifier (use once per page) -->
<div id="main-content"></div>

<!-- class: Multiple classes separated by spaces -->
<p class="highlight important text-center"></p>

<!-- style: Inline CSS (use sparingly) -->
<p style="color: red; font-size: 18px;">Red text</p>

<!-- title: Tooltip text -->
<abbr title="HyperText Markup Language">HTML</abbr>

<!-- data-* : Custom data for JavaScript -->
<button data-product-id="123" data-price="29.99">Add to Cart</button>

<!-- Boolean attributes (no value needed) -->
<input type="text" disabled>
<input type="checkbox" checked>
<video src="movie.mp4" autoplay muted loop></video>
üí° Boolean Attributes:

Some attributes like disabled, checked, required, autoplay don't need a value. Their presence alone activates them. Writing disabled="disabled" or just disabled has the same effect.
